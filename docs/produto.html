Área 51 OIDC Platform

Autenticação federada aplicável rapidamente sobre baseline documentada.

Contexto da necessidade

Órgãos públicos e empresas utilizam OpenID Connect (OIDC) para proteção de acesso, mas frequentemente enfrentam:

  • Pontos de falha onde a montagem manual de Keycloak deixa inconsistências entre equipamentos;
  • Espaço onde verificações obrigatórias PKCE/JWT ficam apenas em papel de projeto;
  • Perda de reproducibilidade entre laboratório, homologação e produção;
  • Calendários longamente consumidos apenas por preparação de infraestrutura.

Linha institucional

O ecossistema Área 51 associa automatização declarada à documentação institucional, reduzindo o custo inicial de adoção quando comparado apenas a projetos inteiramente manuais.

Objetivo de referência temporal: provisionar cenário típico de laboratório em poucos minutos após saneamento das premissas técnicas; maturidade plena para produção exige ciclo próprio da organização cliente.

Camadas funcionais

Baseline em scripts

Configuração OIDC sob controle de versão, compatível OAuth 2.1 e PKCE obrigatório.

Implantação assistida por rotina

Uma entrada de comando alinha Keycloak e aplicação protegida em Docker.

Validação automatizada sugerida

Roteiros de teste conferem comportamento esperado antes de promover cenários produtivos.

Caderno técnico em português

Estrutura para setup, revisão conceitual e encaminhamento de incidentes comuns.

Referência rápida de comandos

Pré-requisitos exemplo: distribuição GNU/Linux atual, Docker ativo e Python compatível declarado pelo repositório.

# Clonar baseline
git clone https://github.com/chmulato/caracore-area51.git
cd caracore-area51

# Dependências declaradas pela release ativa
pip install -r operational/requirements.txt

# Rotina exemplo (ajustar endereços e credenciais)
python operational/setup_oidc_environment.py \
  --server 192.168.1.100 \
  --docker-user admin \
  --docker-pass password

python operational/tests/validate_oidc_flow.py \
  --keycloak-url http://192.168.1.100:8080

Saída esperada quando o cenário fecha sem bloqueios: servidor de identidades ativo com aplicação protegida e validação automatizada bem-sucedida.
